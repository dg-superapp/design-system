# Architecture

**Analysis Date:** 2026-04-24

## Pattern Overview

**Overall:** Two-surface monorepo — a **static design-system guideline** (`project/`) plus a **single-page interactive editor** (`builder/`) that consumes the guideline as a read-only source of truth.

**Key Characteristics:**
- Token source-of-truth is CSS custom properties declared on `:root` in `project/colors_and_type.css`; everything else derives from it.
- `builder/` renders a live editor + 6 preview stages; all preview HTML is generated by template-string renderers in `builder/app.js` writing into one `#stage` container.
- `builder/` references `project/` via relative paths (`../project/assets/*.svg`) and never writes back — the guideline is immutable from the editor's perspective.
- No build step, no framework runtime: vanilla HTML/CSS/JS plus a CDN copy of `lucide` for icons.

## Surfaces

**`project/` — Static Guideline (read-only source of truth):**
- Purpose: canonical definitions of tokens, components, and usage guidance.
- Contains: design-token CSS (`colors_and_type.css`), narrative docs (`README.md`, `ICONOGRAPHY.md`, `SKILL.md`, `SUPERAPP_HOST.md`), per-component HTML specimens under `preview/`, a React/JSX MiniApp UI kit under `ui_kits/miniapp/`, brand assets (`assets/`, `fonts/`, `uploads/`), and audit screenshots (`screenshots/`).
- Consumed by: `builder/` (via `../project/...` URLs) and any downstream MiniApp consuming the kit.

**`builder/` — Interactive Editor (derivative, read-only toward `project/`):**
- Purpose: lets a designer tweak tokens live, preview across 6 contexts, and export updated tokens.
- Contains: `index.html` (shell + editor tab markup), `styles.css` (editor chrome + all component specimen styles driven entirely by `var(--...)` tokens), `app.js` (seed tokens → render editors → render stage), and `tests/` (Playwright smoke + full harness).
- Never mutates files under `project/`; edits live only in memory + exported blobs.

## Token Flow

Source → CSS variables → rendered preview:

1. **Seed.** On load, `app.js` reads the initial token values hard-coded to match `project/colors_and_type.css` (palette, type scale, 4pt spacing scale, radii, shadows) into `COLORS`, `TYPE`, `SPACES`, `RADII`, `SHADOWS` arrays.
2. **Apply.** `apply(key, value)` writes each token to `document.documentElement.style` as a CSS custom property (e.g. `--color-primary`, `--space-3`, `--radius-md`, `--text-body`).
3. **Cascade.** `builder/styles.css` consumes every token as `var(--space-N)`, `var(--radius-*)`, `var(--text-*)`, `var(--color-*)`, `var(--shadow-*)` — so a single `apply()` call instantly re-styles every specimen on screen.
4. **Re-render.** Each editor input handler calls `renderXxx()` + `renderTokens()` + `renderStage()` to refresh the editor's own visuals, the live CSS/JSON token dump, and the active preview stage.
5. **Export.** `#exportBtn` serializes the current token set to CSS and JSON (`#tokenCss`, `#tokenJson`) for copy/download; `#resetBtn` restores guideline defaults via `renderAll()`.

## Editor Panels (7)

Declared as tab buttons in `builder/index.html` (`.editor__tabs` / `[data-panel]`):

1. **brand** — name, tagline, description, light/dark logo upload, clear-space slider.
2. **color** — swatch list; each `<input type="color">` fires `apply()` → `renderColors()` → `renderTokens()` → `renderStage()`.
3. **type** — Latin & Khmer family selects + per-step size range inputs.
4. **space** — 4pt scale editors (`--space-0` … `--space-9`).
5. **radius** — corner-radius scale (`--radius-sm`, `--radius-md`, …).
6. **shadow** — elevation strings (text inputs) feeding `--shadow-*`.
7. **tokens** — read-only CSS + JSON dump with **Copy CSS** / **Copy JSON** buttons and a "Skipped / unsupported" list for non-token CSS encountered during parse.

## Preview Stages (6)

`let currentView = 'guide';` is the selector state. Switching a `.mode` button sets `currentView` and calls `renderStage()`, which dispatches to one function in the `stageRenderers` object in `builder/app.js`:

```js
function renderStage(){ stageRenderers[currentView](); if(window.lucide) lucide.createIcons(); }
const stageRenderers = {
  guide(){ /* brand + palette + type + space/radius/shadow specimens */ },
  components(){ /* 40+ component specimens grouped by category */ },
  landing(){ /* marketing-style landing composition */ },
  dashboard(){ /* KPI cards, lists, nav */ },
  form(){ /* inputs, selection, step indicator, actions */ },
  mobile(){ /* phone-shaped .mobile-frame composition */ }
};
```

Every renderer writes a full `innerHTML` string into `#stage`, then `lucide.createIcons()` hydrates `<i data-lucide="…">` placeholders. No diffing, no virtual DOM — the container is rebuilt each time because all styling is token-driven via CSS, so rebuilds are cheap and always in sync.

## Component Categories (`components` stage, 40+ specimens)

Rendered inline in `stageRenderers.components()`, grouped by category with headings that cite the corresponding `project/preview/*.html` specimen file:

- **Headers / brand chrome** — app header (3-slot navy, drawer+2-actions+badges), hero banner, section header, superapp-host strip (`app-header.html`, `hero-banner.html`, `section-header.html`, `superapp-host.html`).
- **Buttons** — primary, secondary, ghost, destructive, icon-only, segmented tabs (`buttons-primary.html`, `buttons-secondary.html`, `segmented-tabs.html`).
- **Inputs / selection** — text inputs, textarea, select, checkbox, radio, switch, step indicator (`inputs.html`, `selection.html`, `step-indicator.html`).
- **File uploader / viewer** — drop zone, uploading card, file list row, document list row, doc viewer chrome (`file-uploader.html`, `file-list.html`, `document-list-row.html`, `doc-viewer.html`).
- **Navigation** — nav row, side drawer, tabs, ministry picker (`nav-row.html`, `side-drawer.html`, `tabs.html`, `ministry-picker.html`).
- **Lists / data** — list rows, service grid, timestamp tags (`list-rows.html`, `service-grid.html`, `timestamp-tags.html`).
- **Feedback** — alerts, modal, empty state, toast (`alerts.html`, `modal.html`, `empty-state.html`).

Each specimen uses the same class names as the MiniApp UI kit (`.appbar`, `.btn`, `.card`, `.file-card`, `.list-row`, …) defined in `builder/styles.css`.

## Viewport Toggle

A `#viewportMobile` checkbox in `.preview__bar` (see `builder/index.html`) drives:

```js
$('#viewportMobile').addEventListener('change', e => {
  $('#stage').classList.toggle('is-mobile', e.target.checked);
});
```

`#stage.is-mobile` in `builder/styles.css` constrains max-width to a phone breakpoint, letting designers validate any of the 6 stages at mobile width without switching views. The dedicated `mobile` stage further wraps content in a `.mobile-frame` bezel for a device-accurate composition preview.

## Reset / Export Flow

- **Reset** (`#resetBtn`): re-seeds the in-memory token arrays from the built-in defaults (which mirror `project/colors_and_type.css`), clears uploaded logos back to `../project/assets/superapp-logo.svg`, then calls `renderAll()` → all editors + stage repaint → toast "Reset to guideline defaults".
- **Export tokens** (`#exportBtn`): rebuilds the CSS custom-property block and JSON equivalent from the current in-memory state and surfaces them in the **Tokens** panel's `<pre>` blocks; **Copy CSS** / **Copy JSON** push to clipboard.

## Entry Points

- **`builder/index.html`** — loaded directly in a browser; no bundler. Pulls Google Fonts + `lucide` via CDN, then `styles.css` and `app.js`.
- **`project/preview/*.html`** — each file is a self-contained preview you can open standalone; each pulls `../colors_and_type.css` and shared preview styles.
- **`project/ui_kits/miniapp/index.html`** — kit showcase; imports the JSX sources (`app.jsx`, `components.jsx`, `tiles.jsx`) via a browser JSX runtime.
- **`builder/tests/smoke.mjs` / `full.mjs`** — Playwright / `claude-in-chrome` MCP-driven checks run against the builder.

## Cross-Cutting Concerns

- **Icons:** `lucide` CDN; re-hydrated after every `renderStage()` because each render overwrites `innerHTML`.
- **Localization:** a `#lang` toggle (Khmer) in the preview bar; Khmer copy is interleaved inline in specimen strings — no i18n framework.
- **Theming:** `#themeToggle` flips a body class consumed by `builder/styles.css` dark-mode rules; tokens remain the same, only surface/text variables swap.
- **Error surfacing:** non-fatal parse warnings (e.g. unsupported CSS during import) are appended to `#skippedList` in the Tokens panel.

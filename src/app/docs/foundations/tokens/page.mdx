# Design Tokens

Foundations of the **DGC MiniApp Design System**. Every token here ships in the
single registry item `dgc-theme` and is consumed verbatim by every downstream
component (R3.5).

## Install

```bash
npx shadcn@latest add http://registry.016910804.xyz/r/dgc-theme.json
```

One command rewrites your consumer app's `src/app/globals.css` with the full
DGC token set — color scale, semantic aliases, type scale, spacing, radii,
shadows, and the `:lang(km)` Khmer font cascade.

## D1 callout — `--accent` is now `--brand`

The legacy DGC palette exported `--accent` as the CTA color. The shadcn/Tailwind
v4 convention reserves `--accent` for a muted hover/selection color, so we
renamed the Cambodia-blue CTA token to `--brand`. Use `bg-brand`,
`text-brand-foreground`, `hover:bg-brand-hover`, and `active:bg-brand-press`
for every primary action button.

### Khmer cascade (R3.3)

Any element marked `lang="km"` picks up the Khmer font stack
`Noto Sans Khmer → Kantumruy Pro → Battambang` with `line-height: 1.6`.

**Important (Pitfall 8):** Set `<html lang="en">` by default and annotate
Khmer passages with `lang="km"` — do NOT set `<html lang="km">` because it
forces the Khmer stack on Latin text. Block-level example:

<p lang="km">
  នេះគឺជាអក្សរខ្មែរដែលកំពុងបង្ហាញដោយប្រើពុម្ពអក្សរ Noto Sans Khmer។
</p>

## Dark mode

Toggle by adding class `dark` on `<html>`:

```html
<html lang="en" class="dark">
```

All semantic aliases (`--background`, `--foreground`, `--card`, `--muted`,
`--brand`, `--success`, etc.) flip automatically via the `.dark { ... }` block
in `theme.css`. There is **no** `[data-theme="dark"]` selector anywhere
(Pitfall 5).

## Downstream usage (R3.5)

Every registry component declares `dgc-theme` as a registry dependency so the
shadcn CLI installs tokens transitively on first component install:

```jsonc
{
  "name": "hello",
  "type": "registry:ui",
  "registryDependencies": ["dgc-theme"]
}
```

This is the Phase 2 exit contract — no component ships its own colors.
